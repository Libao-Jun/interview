工程化题的关键不是名词多，而是能否说清：问题是什么、方案如何落地、结果如何验证。

## 这一章关注什么

- 开发规范与质量保障如何协同
- 构建、发布、监控链路如何运转
- 模块化与包设计如何影响产物
- 性能优化如何从指标到方案闭环
- 项目架构如何支持迭代速度和稳定性

## 推荐学习路径

<Steps>

### 前端工程化

理解工程化的演进逻辑和工具链协同：从手工到自动化、构建工具选型、HMR 原理、代码质量防线、多环境配置管理。

→ [前端工程化](/docs/engineering/fe-eng)

### 前端模块化

理解模块系统如何影响打包产物和运行时行为：规范演进、ESM vs CJS 差异、打包机制、Tree Shaking 实践、包设计。

→ [前端模块化](/docs/engineering/fe-modules)

### 性能优化

掌握性能优化的完整闭环：指标定义 → 瓶颈定位 → 方案落地 → 效果验证。涵盖 Core Web Vitals、关键渲染路径、框架级优化、水合策略。

→ [性能优化](/docs/engineering/performance)

### 项目架构与交付

理解业务复杂度如何映射到模块边界：分层架构、微前端选型、Monorepo 策略、前端 CI/CD、组件化设计。

→ [项目架构与交付](/docs/engineering/architecture)

</Steps>
