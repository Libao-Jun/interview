## 这一章的目标

- 能说清浏览器如何把资源变成可交互页面
- 能解释 JavaScript 执行模型，而不是只背 API
- 能从布局、重排、重绘和网络链路推导优化思路
- 能讲透 Vue 和 React 的渲染模型与数据流原理
- 能在跨端场景中准确判断技术边界与取舍

## 推荐学习路径

<Steps>

### HTML 与 CSS

语义化标签、盒模型、Flex/Grid 布局、选择器优先级、重排与重绘。这是面试中最容易被追问"为什么"的部分 —— 把浏览器渲染行为和 CSS 规则绑在一起理解。

### JavaScript 与 TypeScript

闭包、原型链、事件循环、类型系统、泛型与类型守卫。面试官通常会从一个简单的输出题切入，然后逐步追问到执行模型和内存管理，直到触及你的上限。

### 浏览器与网络

从 URL 到渲染全链路、HTTP 缓存策略、TCP/UDP、跨域解决方案。这是"性能优化"和"安全"两类追问的共同地基 —— 浏览器和网络的底层机制决定了你能多深地回答这两类问题。

### Vue 与 React

响应式引擎 vs Fiber 架构、编译优化 vs 并发渲染。选择当前岗位要求的主框架深入阅读，另一篇作为横向对比视野即可。

### 跨端与拓展场景

H5 公众号、微信小程序、uni-app 的架构差异和技术边界。如果最近项目中涉及跨端开发，应优先补足这部分 —— 真实场景题远比纯理论题有说服力。

</Steps>

:::info
基础知识的目标不是"读完"，而是**能用自己的话讲出来**。每读完一篇，试着回答这三个问题：

1. 核心机制是什么？（一句话说清）
2. 为什么会这样设计？（设计动机和取舍）
3. 我在项目中怎么验证 / 应用过它？（绑定真实案例）
:::
