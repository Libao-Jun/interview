## CSS3 有哪些新特性

- 圆角（border-radius）
- 阴影（box-shadow）
- 对文字加特效（text-shadow）
- 线性渐变（gradient）
- 旋转、缩放、定位、倾斜
- 增加了更多的 CSS 选择器
- 多背景 rgba

## inline vs inline-block

- `inline` 元素`无法设置宽高`，且`内外边距只在水平方向`有效；
- `inline-block`: 具备行内元素和块级元素的特性
  - 可`设置宽高`和`内外边距`在所有方向有效。

## position属性

- `static` — 默认值，正常布局，正常占位。
- `relative` — 相对定位，相对于元素自身的原始位置进行偏移，原位置保留。
- `absolute` — 绝对定位，完全脱离文档流，相对于最近的使用 relative 父级或祖先级元素进行偏移，不占位。
- `fixed` — 固定定位，完全脱离文档流，相对于浏览器视口进行偏移，不占位，固定完全不走动。
- `sticky` — 粘性定位，元素根据正常文档流进行定位，然后相对它的最近滚动祖先和近块级祖先进行偏移。

## rem vs em

- rem，相对单位，相对于 `HTML 根元素` 的 `font-size`（字体大小）值
- em，相对单位，相对于`当前元素（或父元素）` 的 `font-size`（字体大小）值

## 盒模型

所有 HTML 元素可以看作盒子，由内到外四层：**Content → Padding → Border → Margin**

### 盒模型分类

- 标准盒模型：`content-box` (W3C 标准盒子模型)
  - width/height = `content-width/height`
- 怪异盒模型：`border-box` (IE 盒子模型)
  - width/height = `content-width/height + padding + border`

:::warning[现代项目推荐css设置]
通过 css 的 `box-sizing` 来自由设定盒子模型
:::

### 草图

1️⃣ W3C 标准盒子模型

```txt
┌──────────────────────────────┐
│         margin               │
│  ┌────────────────────────┐  │
│  │       border           │  │
│  │  ┌──────────────────┐  │  │
│  │  │    padding       │  │  │
│  │  │  ┌────────────┐  │  │  │
│  │  │  │  content   │  │  │  │
│  │  │  └─────┬──────┘  │  │  │
│  │  └────────┼─────────┘  │  │
│  └───────────┼────────────┘  │
└──────────────┼───────────────┘
               ▼
若 content = 200px
则
元素 width = content（200px） = 200px
```

2️⃣ IE 盒子模型

```txt
┌─────────────────────────────┐
│         margin              │
│  ┌───────────────────────┐  │
│  │       border          │  │
│  │  ┌─────────────────┐  │  │
│  │  │    padding      │  │  │
│  │  │  ┌───────────┐  │  │  │
│  │  │  │  content  │  │  │  │
│  │  │  └───────────┘  │  │  │
│  │  └───────┬─────────┘  │  │
│  └──────────┼────────────┘  │
└─────────────┼───────────────┘
              ▼
若 content = 200px，padding = 10px，border = 2px
则
元素 width = content（176） + padding（20） + border（4） = 200px
```
## 伪类与伪元素

1️⃣ 介绍

- 伪类(`:`): 根据元素的状态变化使用 CSS — 根据状态使用CSS
- 伪元素(`::`): 在不需新增 HTML 的情况下新增额外元素 — 新增额外元素

2️⃣ 常见伪类与伪元素

- 伪类
  - `:hover`/`:link`/`:visited`
  - `:disabled`/`:enabled`
  - `:first-child`/`:last-child`
- 伪元素
  - `::before`: 当前元素之前产生一个子元素
  - `::after`: 当前元素之后产生一个子元素
  - `::marker`: 调整列表或摘要元素 (summary) 的项目符号和编号的样式。
  - `::placeholder`: 可以用来设定这个提示文字的样式。
  - `::first-letter`: 第一个字母的特殊的样式

## 实现小于 12px

```css
p {
  font-size: baseSize; /* 缩放参照的字体，基于该字体进行放大缩小 */
  transform: scale(n); /* n缩放比例，基于 baseSize 字体大小 */
  transform-origin: 0 0; /* 从左上角缩放，不然位置会偏 */
}
```
## BFC（块级格式化上下文）

### 触发条件

- 根元素 `<html>`
- `float` 不为 `none`
- `position` 为 `absolute` / `fixed`
- `display` 为 `inline-block` / `flex` / `grid` / `table-cell` / `flow-root`
- `overflow` 不为 `visible`

### BFC 特性

| 特性                     | 应用场景                     |
| ------------------------ | ---------------------------- |
| 内部 Box 垂直排列        | —                            |
| 同一 BFC 内相邻 Margin 重叠 | 避免用同一个 BFC 或改为 Flex  |
| BFC 区域不会与浮动元素重叠 | 实现自适应两栏布局           |
| 计算 BFC 高度时包含浮动子元素 | 清除浮动                  |

## 选择器优先级

优先级从高到低：

1. `!important` — 最高，覆盖一切
2. 内联样式（`style="..."`）— 权重 1,0,0,0
3. ID 选择器 — 权重 0,1,0,0
4. 类 / 伪类 / 属性选择器 — 权重 0,0,1,0
5. 标签 / 伪元素选择器 — 权重 0,0,0,1
6. 通配符 `*` — 权重 0,0,0,0

:::info
相同权重下，后定义的覆盖先定义的（层叠规则）。
:::

## 布局方案

### Flex

| 属性            | 作用                         |
| --------------- | ---------------------------- |
| `display: flex` | 开启弹性布局                 |
| `flex-direction`| 主轴方向：`row` / `column`   |

| `flex-wrap`     | 是否换行                     |

## 水平垂直居中

| 方案                      | 适用场景         | 关键代码                                        |
| ------------------------- | ---------------- | ----------------------------------------------- |
| Flex                      | 通用             | `justify-content: center; align-items: center`  |
| Grid `place-items`        | 通用             | `place-items: center`                           |
| 绝对定位 + transform      | 已知/未知尺寸    | `left: 50%; top: 50%; transform: translate(-50%, -50%)` |

## 响应式设计

- **媒体查询**：`@media (max-width: 768px) { ... }`
- **相对单位**：`rem` / `em` / `vw` / `vh` / `%`
- **弹性布局**：Flex / Grid 自动缩放
- **响应式图片**：`<img srcset="..." sizes="...">`
- **视口设置**：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`

## 重排与重绘

| 操作       | 触发           | 代价 |
| ---------- | -------------- | ---- |
| 重排 (Layout) | 几何属性变化（`width`/`height`/`margin`/`position`） | 高 |
| 重绘 (Paint)  | 视觉属性变化（`color`/`background`/`visibility`）   | 中 |
| 合成 (Composite) | `transform`/`opacity` | 低  |

:::warning
性能优化核心：**减少重排**。使用 `transform` 代替 `top/left`；使用 `will-change` 提升合成层；批量修改 DOM 后一次性读取布局。
:::

## 清除浮动

### 为什么要清除浮动[step]
清除浮动是为了清除使用浮动对元素产生的影响，如浮动的元素高度会塌陷，而高度的塌陷使
之后页面后面的布局不能正常显示。

### 高度塌陷[step]
当父级元素的高度小于浮动子元素的时候， 此时就会出现“高度塌陷”

### 清除浮动方案[step]
- 伪元素清除(使用伪元素)：在需要清除浮动的父元素上添加 clearfix 类名
- 块级空元素(使用空的块级元素)：将这个空的 div 元素插入到需要清除浮动元素的父元素的末尾
- clear(both清除浮动)：在需要清除浮动的父元素上添加 clearfix 类名
- overflow：在需要清除浮动的父元素上添加 `overflow: hidden;` 属性

```css tab="使用伪元素"
/*在需要清除浮动元素的父元素上添加 clearfix 类名*/
.clearfix::after {
  content: "";
  display: table;/*block*/
  clear: both;
}
```

```html tab="使用空块级元素"
<!-- 将这个空的 div 元素插入到需要清除浮动元素的父元素的末尾 -->
<div class="parent">
    <div class="float-left"></div>
    <div class="float-right"></div>
    <div style="clear: both;"></div>
</div>
```
```css tab="both清除浮动"
/* 在需要清除浮动的父元素上添加 clearfix 类名*/
.clearfix:after {
  content: ".";
  display: block;
  height: 0;
  clear: both;
  visibility: hidden;
}
.clearfix {
  display: inline-block;
}
```
```css tab="使用overflow"
/*overflow：在需要清除浮动的父元素上添加 overflow: hidden; 属性*/
.parent {
  overflow: hidden;
}
```

### 注意事项[step]
clear 属性只有块级元素才有效
