## Vuex vs Pinia

| 特性     | Vuex             | Pinia                   |
| -------- | ---------------- | ----------------------- |
| Mutation | 必须有，同步修改 | 已移除，Action 统一处理 |
| 模块化   | 嵌套 modules     | 平铺独立 store          |
| TS 支持  | 较弱             | 原生支持                |
| 动态注册 | 需手动           | 默认动态                |

### Vuex

<Callout type="info" title="Vuex 核心">
- **State**：状态数据 
- **Getter**：派生状态 
- **Mutation**：修改 `State`状态的唯一方法 
- **Action**：同步 + 异步操作（提交 `Mutation`） 
- **modules**：模块化
</Callout>


```js tab="userStore.js"
// store/user.js (Vuex 4)
import { createStore } from "vuex";

// 创建一个独立的模块（推荐）
const userModule = {
  // 1. State：相当于 Pinia 中的 ref 数据
  state: () => ({
    name: "Tom",
  }),

  // 2. Getters：相当于 Pinia 中的 computed
  getters: {
    isAdmin: (state) => state.name === "admin",
  },

  // 3. Mutations：同步修改 state 的唯一途径（必须同步）
  mutations: {
    SET_NAME(state, newName) {
      state.name = newName;
    },
  },

  // 4. Actions：处理异步或批量逻辑，最终调用 mutation
  actions: {
    // 相当于 Pinia 中的 login 方法
    login({ commit }, newName) {
      // 可在此添加异步逻辑，如 API 请求
      // const data = await api.login(newName)
      commit("SET_NAME", newName); // 提交 mutation 修改 state
    },
  },
};

// 创建 Store
const store = createStore({
  modules: {
    user: userModule, // 注册模块
  },
});

export default store;
```

```vue tab="选项式中使用"
<template>
  <div>
    <p>用户名: {{ name }}</p>
    <p>是否管理员: {{ isAdmin }}</p>
    <button @click="login('admin')">登录</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapActions } from "vuex";

export default {
  computed: {
    // 映射 state
    ...mapState(["name"]),
    // 映射 getters
    ...mapGetters(["isAdmin"]),
  },
  methods: {
    // 映射 actions（会自动 dispatch）
    ...mapActions(["login"]),
  },
};
</script>
```

```vue tab="在setup中使用"
<template>
  <div>
    <!-- 通过 $store 访问 -->
    <p>用户名: {{ $store.state.name }}</p>
    <p>是否管理员: {{ $store.getters.isAdmin ? "是" : "否" }}</p>

    <!-- 使用 computed 包装 -->
    <p>用户名(computed): {{ userName }}</p>

    <button @click="handleLogin">登录为 Admin</button>
  </div>
</template>

<script setup>
import { computed } from "vue";
import { useStore } from "vuex";

const store = useStore();

// 需要用 computed 包裹才能保持响应式
const userName = computed(() => store.state.name);
const isAdmin = computed(() => store.getters.isAdmin);

function handleLogin() {
  // 调用 action（注意 dispatch）
  store.dispatch("login", "admin");

  // 或者直接提交 mutation（不推荐）
  // store.commit('SET_NAME', 'admin')
}
</script>
```

### Pinia

<Callout type="info" title="Pinia 核心">

- **State**：状态数据 
- **Getter**：派生状态 
- **Action**：同步 +  异步操作（不再区分 mutation）

</Callout>

```ts tab="useUserStore"
export const useUserStore = defineStore("user", () => {
  const name = ref("Tom");
  const isAdmin = computed(() => name.value === "admin");

  function login(newName: string) {
    name.value = newName;
  }

  return { name, isAdmin, login };
});
```

```ts tab="使用"
const store = useUserStore();

// ❌ 错误：直接解构会丢失响应式
const { name } = store;

// ✅ 正确：使用 storeToRefs 保持响应式
const { name } = storeToRefs(store);

// ✅ actions 可以直接解构（不需要 storeToRefs）
const { login } = userStore;
```

<Callout type="info" title="Pinia 定义 Store方式">

- 组合式 `Store`
- 选项式 `Store`

</Callout>


### 核心区别速查表

| 操作                | Pinia                           | Vuex 4                             |
| :------------------ | :------------------------------ | :--------------------------------- |
| **获取 store 实例** | `useUserStore()`                | `useStore()`                       |
| **读取 state**      | `store.name`                    | `store.state.name`                 |
| **读取 getter**     | `store.isAdmin`                 | `store.getters.isAdmin`            |
| **解构保持响应式**  | 用 `storeToRefs`                | 用 `computed` 包裹                 |
| **修改数据**        | `store.login('admin')`          | `store.dispatch('login', 'admin')` |
| **映射辅助函数**    | `mapState(useUserStore, [...])` | `mapState(['name'])`               |
