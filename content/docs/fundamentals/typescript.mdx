## 类型系统概览

| 分类       | 示例                                                   |
| ---------- | ------------------------------------------------------ |
| 基本类型   | `string` / `number` / `boolean` / `null` / `undefined` |
| 对象类型   | `object` / `{ name: string }` / `interface`            |
| 联合类型   | `string \| number`                                     |
| 交叉类型   | `A & B`                                                |
| 字面量类型 | `'left' \| 'right' \| 'center'`                        |
| 泛型       | `T` / `T extends U` / `T = Default`                    |

## any/unknown/never

- `any`：表示关闭类型检查，可以赋值给任何类型，任何类型也可以赋值给它
- `unknown`：表示未知类型，不能直接赋值给其他类型，需要类型收窄
- `never`：表示永不存在的值，空类型，什么都不能赋值给它，它只能赋值给自身。

## interface vs type

TypeScript 中定义类型的两种方式：`接口（interface）`，`类型别名（type alias）`

1️⃣ 核心区别一览

| 维度                        | interface                   | type                       |
| --------------------------- | --------------------------- | -------------------------- |
| 定义对象/契约               | ✅ 推荐                     | ✅ 可以                    |
| extends 继承                | ✅ 多继承                   | ✅ 通过交叉类型（`&`）模拟 |
| implements（类实现）        | ✅                          | ✅（仅对象形态）           |
| 声明合并（同名自动合并）    | ✅ 类型一致合并，不一致报错 | ❌                         |
| 联合类型(`\|`)              | ❌                          | ✅ `A \| B`                |
| 交叉类型(`&`)               | ❌                          | ✅ `A & B`                 |
| 元组                        | ❌                          | ✅ `[string, number]`      |
| 基础类型别名                | ❌                          | ✅ `type ID = string`      |
| 条件类型 / 映射类型 / infer | ❌                          | ✅                         |

```js
1. 扩展方式：interface 可被重复声明，自动合并；type 不可重复声明。
2. 表达能力：interface 主要用于描述对象结构；type 可以表示任何类型，如联合类型、元组。
3. 继承/交叉：interface 用 extends 继承，更易读且性能略优；type 用交叉类型 & 实现类似继承。

⚠️ interface 可被重复声明，自动合并，其自动合并规则如下
- 非函数成员的合并规则（遵循`唯一性`和`类型一致性`原则）
  - 成员必须唯一：若出现同名的非函数成员，它们必须保持类型一致。
  - 类型冲突报错：若同名成员的类型不同，TypeScript 编译器会直接报错。
- 函数成员的合并规则（重载）
  - 后声明的优先级更高
  - 字面量类型有特殊优先级
```

2️⃣ 选择策略

- `interface` 适合定义对象（可扩展、可合并）
- `type` 适合定义类型表达式（联合、交叉、复杂运算）。

## 泛型

### 基本用法

```ts
function identity<T>(value: T): T {
  return value;
}

// 泛型约束
function getLength<T extends { length: number }>(arg: T): number {
  return arg.length;
}

// 默认类型
interface PaginatedResponse<T = User> {
  data: T[];
  total: number;
}
```

### 泛型的意义

<Callout title="核心价值">
  让类型参数化，在保持复用能力的同时保留类型信息。
</Callout>

1️⃣ 没有泛型的问题

```ts
// 方案1：写死类型 → 不通用
function firstString(arr: string[]): string {
  return arr[0];
}

// 方案2：用 any → 丢失类型信息
function firstAny(arr: any[]): any {
  return arr[0];
}
const result = firstAny([1, 2, 3]);
result.toFixed(); // 不报错，但如果传的是字符串数组就会运行时炸
```

2️⃣ 泛型的方式

```ts
function first<T>(arr: T[]): T {
  return arr[0];
}

const a = first([1, 2, 3]); // a 的类型是 number
const b = first(["x", "y"]); // b 的类型是 string
a.toFixed(); // ✅ 有提示
b.toUpperCase(); // ✅ 有提示

// 输入什么类型，输出就保留什么类型，既通用又安全。
```

<Callout title="总结">
  泛型让你写出 `一套逻辑适配多种类型，且每种类型都能被正确推导` 的代码。
</Callout>

### 泛型 vs any vs 联合类型

| 对比         | 泛型        | any         | 联合类型            |
| ------------ | ----------- | ----------- | ------------------- |
| 类型安全     | ✅ 保留     | ❌ 丢失     | ✅ 保留             |
| 输入输出关联 | ✅ 自动推导 | ❌ 断裂     | ❌ 需要手动窄化     |
| 通用性       | ✅ 任意类型 | ✅ 任意类型 | ⚠️ 只能列举有限类型 |

```ts
// any：进去什么出来都是 any，类型链断了
function wrap(v: any): any {
  return { value: v };
}

// 联合类型：能表达有限几种，但输入输出不关联
function wrap(v: string | number): string | number {
  return v;
}

// 泛型：传 string 就返回 string，传 number 就返回 number
function wrap<T>(v: T): { value: T } {
  return { value: v };
}
```

### 泛型 vs 断言

1️⃣ 泛型 vs 断言

| 对比维度     | 泛型                                           | 断言                                    |
| ------------ | ---------------------------------------------- | --------------------------------------- |
| **本质**     | 类型参数化，定义时不预先指定类型，使用时再确定 | 手动覆盖类型推导，绕过自动推导          |
| **方向**     | 类型从调用处**传入**                           | 开发者**指定**类型                      |
| **类型安全** | ✅ 编译时严格检查                              | ⚠️ 绕过当前行**编译时**类型检查，需谨慎 |
| **运行时**   | 完全被擦除，无影响                             | 完全被擦除，无影响                      |
| **使用场景** | 函数/类/接口的复用                             | DOM操作、常量声明、兜底                 |
| **类比**     | 函数的"参数"（类型参数）                       | "强制转换"声明                          |

2️⃣ 断言 vs any

| 对比维度     | `as` 断言                       | `any`                             |
| ------------ | ------------------------------- | --------------------------------- |
| **作用范围** | 单次表达式                      | 整个变量及其传播                  |
| **类型推断** | 断言后仍保留正常检查            | 完全关闭，被赋值给谁谁就污染      |
| **方法提示** | ✅ 有智能提示                   | ❌ 没有，纯编辑器盲写             |
| **危险程度** | ⚠️ 局部风险                     | 🔴 灾难级，污染整个类型系统       |
| **类比**     | 跟编译器说 **这一下要相信我的** | 跟编译器说 **别管了，我自己负责** |

### 常见泛型工具类型

| 工具类型        | 作用                 | 示例                               |
| --------------- | -------------------- | ---------------------------------- |
| `Partial<T>`    | 所有属性变可选       | `Partial<User>`                    |
| `Required<T>`   | 所有属性变必填       | `Required<User>`                   |
| `Readonly<T>`   | 所有属性只读         | `Readonly<User>`                   |
| `Pick<T, K>`    | 选取部分属性         | `Pick<User, 'name' \| 'age'>`      |
| `Omit<T, K>`    | 排除部分属性         | `Omit<User, 'password'>`           |
| `Record<K, V>`  | 构造键值对类型       | `Record<string, number>`           |
| `Exclude<U, E>` | 从联合类型中排除     | `Exclude<'a' \| 'b', 'a'>` → `'b'` |
| `Extract<U, E>` | 从联合类型中提取     | `Extract<'a' \| 'b', 'a'>` → `'a'` |
| `ReturnType<T>` | 获取函数返回值类型   | `ReturnType<typeof fn>`            |
| `Parameters<T>` | 获取函数参数类型元组 | `Parameters<typeof fn>`            |

## 条件类型

```ts
// 基本形式
type IsString<T> = T extends string ? true : false;

// 条件分发（对联合类型逐个判断）
type ToArray<T> = T extends any ? T[] : never;
type Result = ToArray<string | number>; // string[] | number[]

// 阻止分发：用 [T]
type NoDistribute<T> = [T] extends [any] ? T[] : never;
```

### infer 关键字

在条件类型中推断子类型：

```ts
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;
type Result = UnpackPromise<Promise<string>>; // string
```

## 映射类型

```ts
// 基本映射
type Optional<T> = {
  [K in keyof T]?: T[K];
};

// 键的重映射（TS 4.1+）
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};
```

### 函数重载

<Callout title="如何实现函数重载？">
  提供多个函数签名声明，最后给出一个实现签名，但实现代码需要自行做类型守卫判断。
</Callout>

📙 Example

```ts
// 函数签名声明
function greet(person: string): string;
function greet(person: string[]): string[];

// 实现签名 (不是公共签名，对外不可见)
function greet(person: string | string[]): string | string[] {
  if (typeof person === "string") {
    return `Hello, ${person}`;
  }
  return person.map((p) => `Hello, ${p}`);
}
```

<Callout type="warn" title="注意">
  实现签名必须兼容所有重载签名，但不对外暴露，调用时只能匹配重载签名之一。
</Callout>

## 类型守卫

<Callout tittle="概念">
  类型守卫的核心原理是`类型缩窄（Type Narrowing`。 通过条件判断，TypeScript
  会`自动将联合类型缩小为具体的类型`。
</Callout>

| 方式           | 示例                    | 说明                           |
| -------------- | ----------------------- | ------------------------------ |
| `typeof`       | `typeof x === 'string'` | 适用于原始类型的检查           |
| `instanceof`   | `x instanceof Date`     | 检查对象是否是特定类的实例     |
| `in`           | `'name' in obj`         | 检查对象是否包含某个属性       |
| 真值缩小       | —                       | 利用 JS 的真值判断进行类型收窄 |
| 可辨识联合     | —                       | 通过标识字段区分类型           |
| 自定义类型守卫 | `value is Type`         | 实现灵活的类检查               |

<Callout type="warn" title="注意事项">

- 类型守卫必须在条件分支中使用：只有在使用类型守卫进行条件判断后，TS 才会进行类型缩小
- 返回类型必须是类型谓词：自定义类型守卫的返回类型必须是 `value is Type` 格式
- 可辨识联合是最佳实践：对于复杂的联合类型，建议使用可辨识联合模式
- 注意类型收窄的完整性：使用 switch 语句时，建议处理所有可能的分支

</Callout>

## 枚举

| 类型         | 编译产物     | 反向映射 | 场景       |
| ------------ | ------------ | -------- | ---------- |
| 数字枚举     | 双向映射对象 | 有       | 默认       |
| 字符串枚举   | 单向映射对象 | 无       | 可读性高   |
| `const enum` | 内联替换     | 无       | 追求零开销 |

<Callout type="warn">
  `const enum` 在 `isolatedModules`
  模式下有跨模块风险，建议用字符串字面量联合类型替代。
</Callout>

## 类型兼容性

### 结构化类型系统

TS 采用**结构化类型**（鸭子类型），只要形状匹配就兼容，不看声明来源。

```ts
interface A {
  name: string;
}
interface B {
  name: string;
}
const a: A = { name: "x" };
const b: B = a; // OK — 结构相同
```

### 协变与逆变

| 位置   | 方向 | 示例                                              |
| ------ | ---- | ------------------------------------------------- |
| 返回值 | 协变 | `() => Dog` 可赋值给 `() => Animal`               |
| 参数   | 逆变 | `(a: Animal) => void` 可赋值给 `(d: Dog) => void` |
| 双向   | 双变 | `strictFunctionTypes: false` 时参数双变           |

## 声明文件

- `.d.ts` 只包含类型声明，不产生运行时代码
- `declare module 'xxx'` 声明外部模块
- `declare global` 扩展全局类型
- `@types/xxx` 社区维护的类型定义（DefinitelyTyped）

## 常见面试手写题

### 实现 DeepPartial

```ts
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepPartial<T[K]>
    : T[K];
};
```

### 实现 DeepReadonly

```ts
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};
```
