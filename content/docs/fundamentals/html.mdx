## HTML 语义化

### 参考资料

- [HTML 最新规范](https://htmlspecs.com/)：现行标准

<Callout type="warn" title="备注">
- 在 2019 年之前，W3C 发布的是带有版本号的 HTML5 标准。
- 自 2019 年 5 月 28 日起，W3C 宣布 [WHATWG](html.spec.whatwg.org/) 动态标准为 HTML 的唯一版本。
</Callout>

### 对 HTML 语义化的理解

- 用正确的标签做正确的事情
- 让页面的内容结构化，结构更清晰
- 便于对浏览器、搜索引擎解析
- 即使在没有样式 CSS 情况下也以一种文档格式显示，并且是容易阅读的
- 搜索引擎的爬虫也依赖于 HTML 标记来确定上下文和各个关键字的权重，利于 SEO
- 使阅读源代码的人对网站更容易将网站分块，便于阅读维护理解

### 常见语义化标签

目前约有 100 多项语义化标签：

| 分类     | 标签                                             |
| -------- | ------------------------------------------------ |
| 结构标签 | `header` / `nav` / `aside` / `main` / `footer`  |
| 标题标签 | `h1`~`h6` / `hgroup`                            |
| 内容标签 | `article` / `section` / `details` / `time`      |
| 媒体标签 | `figure` / `figcaption`                          |

### 为什么要使用语义化标签

- **SEO**：让网页更容易被搜寻到
- **开发**：提升代码可读性、帮助理解网页架构、减少差异化
- **无障碍**：方便其他设备解析内容（HTML accessibility）

## src vs href

| 属性  | 用途           | 加载方式       | 阻塞行为                          | 典型标签               |
| ----- | -------------- | -------------- | --------------------------------- | ---------------------- |
| `src` | 引入/嵌入资源  | 下载并应用到当前位置 | 普通 `<script>` 阻塞；<br/>`async`/`defer`/`<img>`/`<iframe>` 不阻塞 | `<script>` `<img>` `<iframe>` |
| `href`| 建立链接/关联  | 并行下载，不阻塞解析 | 不阻塞                            | `<a>` `<link>`         |

## data-* 属性

<Callout type="warn" title="应用场景">
- 适用于**小规模配置数据**和**组件状态**管理
- 不适合**存储大量**或**敏感数据**。
</Callout>

### 语法

```html
<element data-*="somevalue">
```

### data-* 属性组成
- 属性名不要包含大写字母（即全小写：包括**小写字母、数字、连字符（-）、下划线（_）、点（.）**）
- 在 `data-` 后必须至少有一个字符，该属性可以是任何字符串

<Callout type="warn" title="注意">
自定义属性前缀 `data-` 会被客户端忽略。
</Callout>

### 命名转换规则

| HTML 属性        | JS 访问方式         | 说明                                 |
| ---------------- | ------------------- | ------------------------------------ |
| `data-user-name` | `dataset.userName`  | 连字符 → 驼峰                        |
| `data-User-Name` | `dataset.username`  | 大写自动转小写（不推荐）             |
| `data-user_id`   | `dataset.user_id`   | 下划线保留，后续连字符转驼峰         |
| `data-user.name` | `dataset.user.name` | 点号保留，原样访问（注意是对象嵌套） |

<Callout title="完整规则总结">
- 连字符（`-`） → 移除，后一个字母转大写（驼峰）
- 下划线（`_`） → 保留不变
- 点号（`.`） → 保留不变
- 大写字母 → 自动转为小写
</Callout>
