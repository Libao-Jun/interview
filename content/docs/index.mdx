| 专题 | 定位 | 核心关注点 |
| --- | --- | --- |
| **基础知识** | 底层打底 | 语义、机制、为什么这样设计 |
| **框架专题** | 进阶深入 | 响应式、渲染管线、状态流、编译优化 |
| **技术栈&灵魂伴侣** | 选型决策 | 生态全景、横向对比、技术选型速查 |

## 各专题内容地图

### 基础知识

先补足能支撑大多数前端追问的底层知识。

| 章节 | 核心能力 |
| --- | --- |
| **HTML** | 语义标签、音视频、Canvas/SVG、Web Storage、Web Worker |
| **CSS** | 盒模型、BFC、选择器优先级、Flex/Grid、重排重绘、响应式 |
| **ES6+** | 变量声明、解构、箭头函数、Promise、模块化、迭代器与生成器 |
| **JavaScript** | 数据类型、原型链、闭包、this、事件循环、异步编程 |
| **TypeScript** | 类型体操、泛型约束、条件类型、映射类型、类型守卫 |
| **Vue 基础** | 响应式原理、组件化、生命周期、路由、状态管理 |
| **Network** | HTTP 演进、TCP/UDP、缓存策略、WebSocket、跨域方案 |
| **浏览器** | 渲染管线、V8 引擎、垃圾回收、跨域方案、Web 安全 |

### 框架专题

用"渲染模型 + 状态流 + 性能"组织 Vue 与 React 面试题。

| 章节 | 核心关注点 |
| --- | --- |
| **Vue 专题** | 响应式引擎（Proxy vs defineProperty）、编译优化（PatchFlag/Block Tree）、渲染管线、组合式函数 |
| **React 专题** | Fiber 架构、Hooks 机制、并发渲染、状态管理对比 |

### 技术栈&灵魂伴侣

从框架选型到工程实践，构建前端知识体系的全景地图。

| 章节 | 核心关注点 |
| --- | --- |
| **Vue 3** | 脚手架、全栈框架（Nuxt）、状态管理（Pinia）、UI 组件库、跨端方案、技术选型速查 |
| **React** | Web 架构（Next.js）、React Native 跨端、状态管理、构建工具、技术选型速查 |
| **Svelte** | Runes 响应式、SvelteKit 全栈、编译时优化、生态工具链 |
| **框架横评** | 五大框架设计哲学、性能基准、生态成熟度、学习曲线、选型决策框架 |

### 工程化与场景题

让"做过项目"真正转化为可验证的面试表达。

| 章节 | 核心关注点 |
| --- | --- |
| **前端工程化** | 开发规范、构建流程、质量保障 |
| **前端模块化** | 规范演进、打包机制、Tree Shaking |
| **性能优化** | 指标定义、瓶颈定位、方案落地 |

## 推荐阅读路径

1. **先看刷题策略**，明确复习节奏和输出目标 —— 知道"怎么复习"比"复习什么"更重要。
2. **基础知识打底**，补齐底层机制的理解 —— 面试追问 80% 来自这里。
3. **框架专题进阶**，理解渲染模型和状态流 —— 这是区分"用过"和"懂了"的分界线。
4. **技术栈选型决策**，建立横向对比视野 —— 选型题考的不是偏好，而是对权衡的理解。
5. **工程化场景实战**，把项目经验组织成可验证的回答 —— 这是最容易拉开差距的环节。

<Callout>

每读完一篇，把结论改写成你自己的回答版本。知识库提供的是**框架和线索**，真正的面试输出需要你用自己的语言填入细节。

</Callout>
