面试准备最怕两件事：一是知识点很多，但没有组织；二是背了很多答案，却和项目经验脱节。这篇策略的目标是让你在打开知识库的任何一页之前，先看清"怎么复习"——用最小的时间投入，把零散知识点转化为面试中的稳定输出。

## 先定位：你当前在哪个阶段

这个知识库覆盖了从初级到专家的四条成长路线，每条路线对应不同的侧重点：

| 岗位 | 核心关注点 | 优先阅读 |
| --- | --- | --- |
| **初级前端** | 浏览器、JavaScript、CSS 和网络链路的稳定输出 | [基础知识](/docs/fundamentals)：HTML → CSS → JavaScript → 浏览器 → Network |
| **中级前端** | Vue / React 渲染更新、状态流和常见项目场景 | [Vue](/docs/fundamentals/vue) / [React](/docs/fundamentals/react) + [状态管理](/docs/fundamentals/store) |
| **高级前端** | 性能、架构和复杂场景题的完整链路 | [工程化与场景题](/docs/engineering)：性能优化 → 项目架构 → 前端工程化 |
| **专家前端** | 跨团队系统性问题、技术演进、稳定性 SLA | [项目架构与交付](/docs/engineering/architecture) + [前端工程化](/docs/engineering/fe-eng) + 跨端全景 |

先锁定你的目标岗位，然后按对应路线集中投入 —— 不要试图把所有内容都读完。

## 三轮准备法

### 第一轮：建立地图（1-2 天）

目标不是记住细节，而是知道"有哪些话题"和"大概在哪个位置"。

- 打开[知识库总览](/docs)，快速浏览基础知识 15 篇 + 工程化 5 篇的标题和描述。
- 每看完一篇，只记录一句话：这篇解决什么面试问题。
- 画出你的知识地图：哪些已经能说清、哪些只有模糊印象、哪些完全没概念。

这一轮结束后，你应该能回答："如果面试官问 X，我大概知道去哪找答案。"

### 第二轮：补齐回答骨架（核心阶段，占 60% 时间）

这是最关键的一轮 —— 把"知道"转化为"能说出来"。

每篇文档按这个顺序组织你的回答：

1. **先给一句结论**（30 秒内说清这是什么）
2. **再解释核心原理**（2-3 分钟展开机制和过程）
3. **最后补一个项目场景**（绑定你的真实项目中的取舍或验证）

以 JavaScript 事件循环为例：

| 层次 | 内容 |
| --- | --- |
| 结论 | "事件循环是 JavaScript 的异步调度机制，由宏任务队列和微任务队列组成。" |
| 原理 | 调用栈 → 宏任务 → 微任务清空 → 渲染管线 → 下一轮循环 |
| 项目 | "在我们的中台项目中，利用微任务优先的特性优化了批量 DOM 更新的时序。" |

:::info
不要试图逐篇精背，而是**每篇提炼出一个可复述的 3 层骨架**。知识库的正文提供的是素材和线索，你需要用自己的语言填入细节。
:::

### 第三轮：模拟追问（占 20% 时间）

每个知识点至少继续追问两层，这是区分"背过"和"掌握了"的分界线：

- **第一层追问：为什么会这样？**—— 考察你对设计动机的理解。比如"为什么 Promise 的微任务优先级高于 setTimeout？"→ 这关系到渲染帧之间的执行时机。
- **第二层追问：如果条件变了呢？**—— 考察你的工程判断力。比如"如果 QPS 提高 10 倍，这个方案还成立吗？瓶颈会在哪？"
- **第三层追问：你怎么验证的？**—— 考察你的实战经验。比如"你在项目中用什么工具 / 指标来验证优化效果的？"

| 时间 | 重点 | 对应知识库章节 | 输出要求 |

| **周一** | Web 核心 | [HTML](/docs/fundamentals/html) → [CSS](/docs/fundamentals/css) → [ES6+](/docs/fundamentals/es6) → [JS](/docs/fundamentals/javascript) | 补齐空白术语，确保每个概念能一句话说清 |
| **周二** | 语言进阶 + 浏览器 | [TypeScript](/docs/fundamentals/typescript) + [浏览器](/docs/fundamentals/browser) + [Network](/docs/fundamentals/network) | 从 URL 到渲染全链路能连贯复述 |

| **周末** | 模拟面试 | 回顾三轮追问的薄弱点 | 找同事做 1 小时模拟，只复盘表达卡顿的地方 |

## 判断准备是否有效

一篇文档读完后，用这四条标准检验你是否真正吸收了：

- ✅ 能在 **1 分钟**内说明核心结论（不翻笔记）
- ✅ 能在 **3 分钟**内展开原理和过程（有逻辑链，不是堆术语）
- ✅ 能补一个项目中的**真实案例或取舍**（不是编的，是自己做过的）
- ✅ 面对追问**不会立刻跳回"背诵模式"**（能顺着追问的逻辑往下推）

如果四条都满足，这篇内容就真正进入了你的面试系统。否则回到第二轮，针对卡顿的地方重新组织回答骨架。
